# AI 一致性质检助手使用说明

## 目标

对当前数据组的原图和结果图进行辅助对比，重点判断结果图中应保留的主体内容是否与原图一致，并提示畸形、结构变形、细节丢失、颜色错误、文字或 Logo 错误、画面瑕疵和版面不协调等问题。

## 使用流程

1. 在质检队列中选择数据组。
2. 工具根据设置自动执行本地初筛，也可以点击“本地AI检测”。
3. 本地结果显示一致性评分、风险等级、建议结论和八个检测维度。
4. 本地结果为中风险或高风险且在线配置完整时，可自动执行在线深度复核。
5. 任意数据组都可手动点击“在线深度复核”。
6. 使用“采纳AI标签”和“采纳AI备注”把建议填入人工质检区，再由质检员修改和确认。
7. 最终仍由质检员点击“质检通过”或“质检不通过”。

## 八个检测维度

- 主体一致性：类别、数量、身份、轮廓和关键识别特征。
- 结构完整性：比例、部件、透视、错位、扭曲和畸形。
- 细节保留：纹理、配件、边缘、图案、标签和小结构。
- 颜色一致性：主色、局部颜色、材质色和异常色偏。
- 文字与 Logo：拼写、字符形态、内容、位置、清晰度和缺失。
- 背景符合性：背景是否符合中英文指令，是否错误保留原背景。
- 画面质量：模糊、重影、脏点、锯齿、破图、融合错误和生成瑕疵。
- 版面协调性：主体占比、遮挡、留白、视觉重心和元素协调。

## 本地初筛边界

本地规则不上传图片。它能够发现明显的结构、颜色、清晰度和版面差异，但不能可靠理解文字拼写、Logo 身份和复杂指令语义；这些维度会显示“未检测”，而不是伪造结论。下载 DINOv2 ONNX 模型后，主体身份判断会增加语义相似度信号。

## 在线设置

点击“AI设置”，选择：

- OpenAI：Base URL 留空使用官方 `/v1/responses`。
- Gemini：Base URL 留空使用官方 `generateContent` 地址。
- OpenAI兼容自定义API：填写完整 Base URL，工具调用 `/chat/completions`。

需要填写模型名称和 API Key。可设置请求超时、上传图片最长边、是否自动本地检测及是否智能触发在线复核。

## 数据与隐私

在线复核会把原图、结果图、中英文指令和本地检测摘要发送给所选服务商。API Key 保存在当前 Windows 用户的应用设置中，不写入项目 AI 日志、操作日志或 Excel 报表。项目中的 `.质检工具/ai_review_log.jsonl` 仅保存检测结果和图片签名，用于恢复当前结果与统计。

## 安全约束

AI 永远不会自动执行：

- 质检通过。
- 质检不通过。
- 删除数据组。
- 移动数据组。
- 覆盖已有人工备注。

AI 任务运行期间，文件流转按钮会暂时禁用，防止后台读取图片时文件被移动或删除。
