# 数据堂质检工具 AI 一致性质检助手设计

## 目标

在现有 Windows 桌面质检工具内新增“本地初筛 + 在线深度复核”的 AI 一致性质检能力，对原图与结果图进行成对比较，辅助发现主体不一致、结构变形、细节丢失、颜色错误、文字或 Logo 错误、背景不符合要求、画面瑕疵和版面不协调等问题。AI 只提供建议、问题标签和返修备注，最终通过、返修和文件移动必须由人工确认。

## 已确认产品决策

- 同时支持本地检测和在线深度复核。
- 在线服务支持 OpenAI、Gemini、OpenAI 兼容自定义 API。
- 默认采用混合触发：选中数据组后先做本地检测；本地为中高风险且在线配置完整时自动复核；始终保留手动在线复核按钮。
- AI 自动生成标签、返修备注和“建议通过 / 建议复核 / 建议返修”，但绝不自动移动文件夹。
- 原有队列、图片查看、指令编辑、通过/返修、撤销、删除、日志和 Excel 导出全部保留。

## 架构

### 本地分析层

`LocalConsistencyReviewer` 使用 Pillow、NumPy 和可选 DINOv2 ONNX 模型，计算语义相似度、结构相似度、颜色相似度、细节保留、清晰度和版面占用等指标。DINOv2 未安装时仍运行规则指标，并将无法可靠本地判断的文字/Logo、背景语义标为“未检测”，不得伪装为已验证。

### 在线复核层

`OnlineReviewClient` 根据设置选择：

- OpenAI 官方 Responses API；
- Gemini generateContent REST；
- OpenAI 兼容 `/chat/completions`。

请求包含原图、结果图、中文指令、英文指令和本地分析摘要，要求返回固定 JSON。解析层清洗 Markdown 代码块、校验范围、归一化问题分类并生成可编辑返修备注。

### 数据与审计

每次本地或在线检测追加写入项目目录 `.质检工具/ai_review_log.jsonl`，包含图片签名、人员、数据组、阶段、提供商、评分、风险、建议和问题列表，不记录 API Key。切换数据组时读取同一图片签名的最新结果，避免重复调用。通过或返修操作日志同时记录当时采用的 AI 摘要，Excel 增加 AI 统计和问题分类。

### 界面

右侧保留原有质检信息，并增加“AI 一致性质检”页签：

- 一致性评分、风险等级、建议结论；
- 8 个检测维度的状态、分数和说明；
- 本地检测、在线深度复核、AI 设置、采纳标签、采纳备注、清空结果；
- 运行状态和错误提示。

AI 设置保存到当前用户的 `QSettings`。API Key 使用密码输入框显示，日志和界面错误不得输出完整 Key。

## 风险判定

- `low`：总风险分低，未出现高置信度关键问题；建议通过或人工快速确认。
- `medium`：存在一项或多项疑似问题；建议复核，并可触发在线复核。
- `high`：主体、结构、文字/Logo 等关键问题达到高置信度，或多个中风险问题叠加；建议返修。

在线结果优先于本地结果展示，但本地指标仍保留在审计日志中。

## 错误处理

- 本地模型缺失：继续规则检测，显示“语义模型未安装”。
- 在线配置不完整：手动按钮给出明确配置提示；自动触发跳过，不阻塞切换数据。
- 超时、HTTP 错误、非 JSON 返回：保留本地结果，显示可重试错误，不自动填写错误标签。
- 数据组切换时旧任务完成：通过图片签名核对，禁止把旧结果写到新数据组。
- 关闭程序时任务运行：提示等待完成，不强制终止写入。

## 测试范围

- 本地指标、风险映射、缺模型降级；
- 三类在线请求负载与响应解析；
- AI 日志追加、去重和签名隔离；
- 操作日志 AI 字段向后兼容；
- Excel AI 统计；
- 主界面按钮、采纳标签/备注和智能触发的无头测试；
- 发布包静态契约和 Python 语法编译。
